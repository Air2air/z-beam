  ]

  <>
    <JsonLD data={schemas} />

### 2. **Category Pages** (Collection Level)
**Path:** `/materials/[category]`  
**Component:** `JsonLD`  
**Schemas:** 5 types via `@graph`
- CollectionPage
- BreadcrumbList (separate for discoverability)
- ItemList (all materials in category)
- Dataset (category-level aggregation with JSON/CSV/TXT formats)
- WebPage

**Example:** `/materials/ceramic`

**Schema Structure:**
```typescript
{
  '@context': 'https://schema.org',
  '@graph': [
    {
      '@type': 'CollectionPage',
      '@id': `${url}#webpage`,
      'breadcrumb': { '@id': `${url}#breadcrumb` },
      'mainEntity': { '@id': `${url}#itemlist` }
    },
    {
      '@type': 'BreadcrumbList',
      '@id': `${url}#breadcrumb`,
      'itemListElement': [...]
    },
    {
      '@type': 'ItemList',
      '@id': `${url}#itemlist`,
      'numberOfItems': count,
      'itemListElement': [...]
    },
    {
      '@type': 'Dataset',
      '@id': `${url}#dataset`,
      'distribution': [JSON, CSV, TXT formats],
      'hasPart': [material datasets]
    },
    {
      '@type': 'WebPage',
      '@id': url,
      'mainEntity': { '@id': `${url}#dataset` }
    }
  ]
}
```

**Status:** ✅ Fully normalized - uses `JsonLD` component with `@graph` pattern

## Component Architecture

### JsonLD Component
**File:** `app/components/JsonLD/JsonLD.tsx`

```typescript
export function JsonLD({ data }: JsonLdProps) {
  const jsonString = JSON.stringify(data).replace(/\\\//g, '/');
  
  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: jsonString }}
    />
  );
}
```

**Features:**
- Renders clean `<script type="application/ld+json">` tags
- Removes escaped forward slashes for cleaner markup
- Works in Next.js App Router with RSC
- Accepts any schema structure (single or `@graph`)

---

### MaterialJsonLD Component
**File:** `app/components/JsonLD/JsonLD.tsx`

```typescript
export function MaterialJsonLD({ article, slug }: { article: any; slug: string }) {
  let jsonLdSchema;
  
  try {
    const factory = new SchemaFactory(article, slug);
    jsonLdSchema = factory.generate();
  } catch (error) {
    jsonLdSchema = createJsonLdForArticle(article, slug);
  }
  
  if (!jsonLdSchema) return null;
  
  const jsonString = JSON.stringify(jsonLdSchema, null, 2).replace(/\\\//g, '/');
  
  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: jsonString }}
    />
  );
}
```

**Features:**
- Specialized for material pages
- Uses SchemaFactory with fallback
- Generates 7-8 schema types
- Pretty-printed JSON (2-space indent)

## Best Practices Implemented

### 1. ✅ @graph Pattern
All multi-schema pages use `@graph` array for proper entity relationships.

### 2. ✅ @id References
Entities reference each other via `@id` for semantic connections:
```json
{
  "breadcrumb": { "@id": "https://www.z-beam.com/materials/ceramic#breadcrumb" }
}
```

### 3. ✅ Consistent URL Structures
All URLs use canonical format without trailing slashes.

### 4. ✅ License Information
Datasets include proper Creative Commons BY 4.0 licensing.

### 5. ✅ Distribution Formats
Datasets provide multiple formats (JSON, CSV, TXT) for accessibility.

### 6. ✅ E-E-A-T Signals
- Author credentials (Person schema)
- Regulatory standards
- Publication/modification dates
- Expert content signals

### 7. ✅ Accessibility
- ARIA labels
- Semantic HTML
- Keyboard navigation
- Alt text for images
