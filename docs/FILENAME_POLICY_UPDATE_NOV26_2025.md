Added explicit parentheses checks:
```javascript
// Check for parentheses in filenames
if (fileName.includes('(') || fileName.includes(')')) {
  this.errors.push({
    type: 'parentheses_in_filename',
    fileName,
    context,
    message: `File name "${fileName}" contains parentheses (not allowed - remove parentheses)`
  });
  this.stats.namingErrors++;
  return false;
}

// Check for parentheses in slugs
if (slug.includes('(') || slug.includes(')')) {
  this.errors.push({
    type: 'parentheses_in_slug',
    slug,
    context,
    message: `Slug "${slug}" contains parentheses (not allowed - remove parentheses from filename)`
  });
  this.stats.namingErrors++;
  return false;
}
```

## Chemical Formula Handling

### Problem
Chemical formulas often use parentheses for clarity:
- Acrylic (PMMA)
- Titanium Alloy (Ti-6Al-4V)
- Silicon Carbide (SiC)

### Solution
Remove parentheses in filenames:
```bash
# Before (with parentheses) ❌
acrylic-(pmma)-laser-cleaning.yaml
titanium-alloy-(ti-6al-4v)-settings.yaml
silicon-carbide-(sic)-laser-cleaning.yaml

# After (without parentheses) ✅
acrylic-pmma-laser-cleaning.yaml
titanium-alloy-ti-6al-4v-settings.yaml
silicon-carbide-sic-laser-cleaning.yaml
```

## Rationale

### Why Prohibit Parentheses?

1. **URL Encoding Issues**
   - Parentheses become `%28` and `%29` in URLs
   - Creates ugly, non-semantic URLs
   - Poor user experience

2. **Slug Validation Failures**
   - Regex pattern `/^[a-z0-9-]+$/` rejects parentheses
   - Causes build failures
   - Blocks git push via pre-push hooks

3. **SEO Penalties**
   - Search engines prefer clean, semantic URLs
   - Encoded characters hurt rankings
   - Reduced crawlability

4. **File System Compatibility**
   - Some file systems have issues with special characters
   - Cross-platform consistency
   - Git-friendly naming

5. **Consistency**
   - Uniform naming convention across all files
   - Easier to find and reference files
   - Reduced cognitive load

Runs automatically before every `git push`:
- Checks all frontmatter files
- Checks all image files

- **Blocks push if parentheses detected**

### Build-Time Validation
```json
{
  "prebuild": "npm run validate:naming && npm run validate:metadata",
  "vercel-build": "npm run validate:naming && next build"

```

Validation runs before every build:
- Ensures all filenames comply
- Prevents deployment of non-compliant files
- Fail-fast approach

## Migration Path for New Files

When adding new materials with chemical formulas:

1. ✅ **DO:** Remove parentheses from filename
   ```
   acrylic-pmma-laser-cleaning.yaml
   titanium-alloy-ti-6al-4v-settings.yaml
   ```

2. ✅ **DO:** Use full chemical name in title/metadata
   ```yaml
   name: acrylic-pmma-laser-cleaning
   title: "Acrylic (PMMA) Laser Cleaning"  # Display name can have parentheses
   ```

3. ❌ **DON'T:** Add parentheses to filename
   ```
   acrylic-(pmma)-laser-cleaning.yaml  # ❌ BLOCKED by validation
   ```
